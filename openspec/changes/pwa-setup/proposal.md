## Why

Anonymix is mobile-first but has no PWA capabilities. Users have to open a browser tab every time — there's no install prompt, no home screen icon, no offline indicator, and no push notification foundation. For a social app where timing matters (deadlines, reveals), the gap between "check if anything happened" and "get notified" is the difference between engagement and abandonment.

This change sets up the PWA foundation: manifest, service worker, install prompt, and notification permission request. Actual push notification delivery (what triggers, what message) is a separate change — this lays the infrastructure.

## What Changes

- **Web app manifest** — name, icons (already exist at `public/icons/`), theme colors, display mode
- **Service worker** via `vite-plugin-pwa` — cache strategy, offline fallback
- **Install prompt** — "Add to Home Screen" banner or button, shown contextually (not on first visit)
- **Notification permission** — request permission at the right moment (after first tape action, not on page load). Store push subscription for future use.
- **Offline indicator** — subtle banner when offline ("You're offline — changes will sync when you reconnect")

## Non-goals

- Push notification delivery/backend (separate change — needs Supabase Edge Functions + Web Push)
- Background sync for offline submissions
- Full offline mode (read-only cache is fine)
- App store listings

## Capabilities

### New Capabilities
- `pwa-shell`: Web manifest, service worker registration, install prompt
- `notification-permission`: Permission request flow with subscription storage

### Modified Capabilities
- `app-scaffold`: Register service worker on app load

## Impact

- `vite-plugin-pwa` dependency
- `public/manifest.webmanifest` (or generated by plugin)
- Service worker generated by Vite plugin
- New component: install prompt UI
- New hook: `useInstallPrompt` (captures `beforeinstallprompt` event)
- New hook: `useNotificationPermission` (request + store subscription)
- Minor: `index.html` meta tags for PWA
