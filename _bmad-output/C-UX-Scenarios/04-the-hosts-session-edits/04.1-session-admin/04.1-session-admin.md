# 04.1-Session Admin

## Page Metadata

| Property | Value |
|----------|-------|
| **Scenario** | 04: The Host's Session Edits |
| **Page Number** | 04.1 |
| **Platform** | Mobile |

## Overview

**Page Purpose:** Manage an ongoing session — but not as a separate admin page. Admin is elevated permissions on existing views: edit mode on the tape crate, a reorder mode, a members view, and session settings. The host never leaves the session to "go to admin."

**User Situation:** A few weeks into running the session, the current tape just wrapped up. They're on the bus checking results, feeling the creative itch to write the next prompt. They also want to rearrange upcoming tapes because a friend suggested a theme that fits better next.

**Mental State:**
- **Hope:** Tweak the session in a couple of taps and get back to being a player, not an admin.
- **Worry:** Managing the session becoming a chore — dreading the overhead of keeping the game alive.

**Discovery Method:** Opens the app from home screen bookmark, navigates to session from session home.

**Entry Context:** Active session → host sees edit affordances on existing views

**Exit Action:** Changes saved — scenario success ✓

## Design Decisions

- **No dedicated admin page:** Admin is permissions, not a destination. Host sees edit affordances (pencil icon, "+" card, gear icon) on the same views all members use.
- **Three modes on the tape crate:**
  - **Browse mode** (01.2) — all members, read-only flip through tapes
  - **Edit mode** — host only. Future tapes are editable (prompt text), "+" to add new tape. Completed/current tapes locked. Reuses crate-flip from 01.2 / 03.1.
  - **Reorder mode** — host only. Top-down view of tape crate, simplified rectangles with truncated prompt text + drag handle. Simple, direct manipulation.
- **Members view:** Separate sheet/overlay accessible from session header. All members can see the member list. Host sees additional controls: invite (copy link), remove member, transfer host role.
- **Session settings:** Name, description, timing (submission/voting windows) accessible from gear icon in session header. Host only.

## Content Needs

- Edit mode: editable prompt text on future tape cards, "+" card to add new tape
- Reorder mode: simplified tape rectangles with truncated prompt + drag handle
- Members view: member list with avatars, host controls (invite, remove, transfer)
- Session settings: name, description, submission window (default 2 days), voting window (default 5 days)

## Driving Forces Addressed

- **Effortless creation (13):** Editing feels like browsing with extra controls — not a separate admin workflow
- **Fear of admin as chore (14):** No admin destination to navigate to. Couple of taps, done. The host is a player first.

## On-Page Interactions (Storyboard Items)

- Edit mode toggle on tape crate (pencil icon or long-press)
- Edit prompt text on future tape cards (inline editing)
- Add new tape via "+" card in the crate
- Switch to reorder mode — top-down drag view
- Members view sheet — invite (copy link), remove, transfer host
- Session settings sheet — edit name, description, timing

## Complexity Notes

- Medium complexity — three crate modes (browse, edit, reorder) on the same component
- Permission-based affordances: same views, host sees more controls
- Members view and settings are sheets/overlays, not separate pages
- Drag reorder on mobile needs careful touch target sizing

## Open Questions

- Can non-host members invite others, or is that host-only?
- Can the host delete future tapes, or only edit/reorder?
- Should there be a "pause session" or "end session" action?
