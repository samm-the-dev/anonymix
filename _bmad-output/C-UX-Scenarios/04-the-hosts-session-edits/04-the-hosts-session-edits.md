# 04: The Host's Session Edits

**Project:** Anonymix
**Created:** 2026-03-24
**Method:** Whiteport Design Studio (WDS)
**Design Intent:** C (Discuss)
**Design Status:** not-started

---

## Transaction (Q1)

**What this scenario covers:**
Manage an ongoing session — add new tapes, rearrange the order, adjust settings, and handle members — without it feeling like admin work.

---

## Business Goal (Q2)

**Goal:** Engagement Quality + Active Participation
**Objective:** Sessions that evolve and stay fresh keep participation high. If the host burns out, the whole game dies.

---

## User & Situation (Q3)

**Persona:** The Host (Tertiary)
**Situation:** A few weeks into running the session, the current tape just wrapped up. They're on the bus checking results, feeling the creative itch to write the next prompt. They also want to rearrange upcoming tapes because a friend suggested a theme that fits better next.

---

## Driving Forces (Q4)

**Hope:** Tweak the session in a couple of taps and get back to being a player, not an admin.

**Worry:** Managing the session becoming a chore — dreading the overhead of keeping the game alive.

---

## Device & Starting Point (Q5 + Q6)

**Device:** Mobile
**Entry:** Opens the app from home screen bookmark, navigates to session admin from within their active session.

---

## Best Outcome (Q7)

**User Success:**
New tape added, order rearranged, took under a minute — still feels like fun, not work.

**Business Success:**
Session stays active over time, host doesn't burn out, game loop sustains across weeks and months.

---

## Shortest Path (Q8)

1. **Session Admin** — Views session settings, adds a new tape, rearranges tape order, manages members if needed ✓
