## Overview

**Page Purpose:** Create tapes with prompts, then wrap them in a session and share the invite link. Tapes first (the fun part), session admin last.

**User Situation:** The organizer friend who just convinced the group to try Anonymix. They're at their desk after sending the group chat message, riding the momentum of "let's do this" — they want to have the session ready before the group's enthusiasm fades.

**Mental State:**
- **Hope:** Get the session live with a few great prompts queued up before the group's excitement cools.
- **Worry:** Setup taking so long or being so confusing that they lose momentum and the whole thing stalls before it starts.

**Discovery Method:** Already in the app — just finished setting up their own account, now taps to create a new session.

**Entry Context:** Post-onboarding → taps create session

**Exit Action:** Invite link copied, session live — scenario success ✓

## Design Decisions

- **Tapes first, session last:** Front-load the creative part (writing prompts), back-load the bookkeeping (name, description, settings). The prompts are exciting, the admin is not.
- **One tape card at a time:** Reuse the crate-flip animation from Tape List (01.2). Flip to add another tape, flip back to edit. Never more than one new/edit card visible at once.
- **Minimum viable session:** One tape with a prompt is enough to create. No pressure to queue multiple — but the flip interaction makes it easy to add more.
- **Session-wide timing:** Submission window (default 2 days) and voting window (default 5 days) set at session level, not per-tape. Host can adjust but doesn't have to.
- **Invite link celebration:** Session creation culminates in a "You're live!" moment with the invite link and native share sheet. This is the Host's payoff.

## Flow

1. **First tape card** — write prompt
2. **Add more?** — flip to a new blank tape card, or advance to session details
3. **Session details** — name, description (optional), timing settings (submission/voting windows with defaults)
4. **Create** → session goes live
5. **Celebration** — "Your session is live!" + invite link + share sheet

## Content Needs

- Tape card: prompt text input
- Crate-flip animation to add/navigate tape cards
- Session details: name, description (optional)
- Timing settings: submission window (default 2 days), voting window (default 5 days)
- Minimal additional settings (e.g., vote point count — host override)
- Create button
- Post-creation: celebration moment, invite link, native share sheet

## Driving Forces Addressed

- **Effortless creation (13):** Tapes first, admin last, smart defaults. One prompt is enough to launch.
- **Fear of admin as chore (14):** Defaults pre-filled, description optional, minimal settings. Under 5 minutes.

## On-Page Interactions (Storyboard Items)

- Write prompt on tape card
- Crate-flip to add another tape card (reuses 01.2 pattern)
- Advance to session details
- Adjust timing settings (optional — defaults work)
- Create session
- Celebration moment + invite link + share sheet

## Complexity Notes

- Medium complexity — multi-step creation flow with card animation
- Reuses crate-flip pattern from Tape List (01.2)
- Smart defaults reduce required decisions to just: write a prompt, name the session, create
- Deferred decisions: pace modes (ML feature), additional session settings
