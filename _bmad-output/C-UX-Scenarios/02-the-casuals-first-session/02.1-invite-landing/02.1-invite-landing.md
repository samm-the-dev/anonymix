# 02.1-Invite Landing

## Page Metadata

| Property | Value |
|----------|-------|
| **Scenario** | 02: The Casual's First Session |
| **Page Number** | 02.1 |
| **Platform** | Mobile |

## Overview

**Page Purpose:** Conversion page — see what you're joining and tap Join. Social proof does the selling, not marketing copy. No scroll on mobile.

**User Situation:** Early 30s, scrolling their phone during a break. A friend just dropped an invite link in the group chat with "we're doing this instead of Music League now." They're curious but not invested — if it's easy they'll join, if it's not they'll say "I'll do it later" and never will.

**Mental State:**
- **Hope:** Be part of the group thing their friends are excited about without it becoming a task.
- **Worry:** Getting stuck in a signup maze — too many screens, too many decisions, having to figure out how to connect their music app.

**Discovery Method:** Taps an invite link shared in a group chat (WhatsApp, iMessage, Discord), opens in mobile browser or PWA.

**Entry Context:** Invite link from group chat → opens in browser/PWA

**Exit Action:**
- **New user:** Taps Join → Auth / Sign Up (02.2)
- **Existing user (logged in):** Could auto-join and land in session, skipping this page entirely
- **Existing user (not logged in):** Taps Join → Auth → auto-joins session

## Design Decisions

- **No-scroll constraint:** Everything must fit on one mobile screen. Tight layout budget — every element must earn its space.
- **Two content variants:**
  - **New user:** Session name, member avatars, first few tape prompts, brief "what is Anonymix" explainer, Join button
  - **Existing user:** Session name, member avatars, first few tape prompts, more session detail in the space freed up by removing explainer (tape history, current status, participation stats)
- **Social proof over exposition:** Member avatars and real prompts show the vibe faster than any explanation.
- **Existing logged-in users:** May skip this page entirely and auto-join on invite link tap.

## Content Needs

- Session name
- Member avatars (who's already joined)
- First few tape prompts (shows the vibe / what kind of prompts this group does)
- New user only: brief "what is Anonymix" explainer (compact)
- Existing user only: additional session info (tape history, current tape status, participation stats)
- Join button — single prominent CTA

## Driving Forces Addressed

- **Effortless participation (14):** Social proof sells it — friends' faces, fun prompts, one button
- **Fear of too many steps (14):** One page, no scroll, one button. See it, want it, join it.

## On-Page Interactions

- None beyond tapping Join. This is a read-and-decide page.

## Complexity Notes

- Low UI complexity but high stakes — this is the conversion gate for every new player
- Two content variants (new vs existing user) — layout must accommodate both without scroll
- Opens in mobile browser from group chat — not the app. PWA install may come later.
