# 01.4-Tape: Playlist Ready

## Overview

**Page Purpose:** Submissions are closed, playlist has been generated — go listen in your music app. Target UX is NOT a separate page but a state change on the tape card in session view.

**Entry Context:** Submission deadline closed, playlist finalized. User gets a notification ("Playlist ready — go listen!") or sees updated tape card status in session view.

**Exit Action:** Taps "Open in [platform]" on tape card → deep-links to playlist in native music app → leaves Anonymix

## Design Decisions

- **Not a page — a card state:** Target UX is a state change on the tape card in session view. Tape card updates to "Playlist ready" with an "Open in Spotify" / "Open in YouTube Music" button. One tap, deep-link, done.
- **Idempotent:** Button should work on repeated taps — always opens the same playlist.
- **Full-page fallback:** If a platform can't support seamless playlist injection (YouTube Music API limitations), a full page may be needed showing the track list with manual options (copy link, open tracks individually). Spec this fallback only if needed during development.
- **Notification-driven:** User likely re-enters via a "Playlist ready" push notification, landing on session view where the tape card shows the updated state.

## Content Needs (Tape Card — Playlist Ready State)

- Updated status: "Playlist ready" or "Go listen"
- Platform-specific CTA button: "Open in Spotify" / "Open in YouTube Music"
- Possibly: track count ("8 songs")

## Driving Forces Addressed

- **Music discovery (15):** This is the bridge to the actual listening — the moment they get to hear what friends picked
- **Friction elimination (15):** One tap to the playlist in their native app. No intermediate pages.

## On-Page Interactions

- Tap CTA → deep-link to playlist in native music app
- No other interactions needed on the card state

## Complexity Notes

- Low UX complexity (it's a card state, not a page) but high technical complexity (playlist generation per platform, deep-linking)
- Technical unknowns: Spotify API playlist creation vs YouTube Music API limitations
- Fallback page only specced if platform constraints require it

## Open Questions

- Spotify playlist creation: add to user's library or standalone playlist link?
- YouTube Music: API support for playlist creation — may need alternative approach
- Notification copy and timing for "playlist ready"
