# 01.1-Session Home

## Page Metadata

| Property | Value |
|----------|-------|
| **Scenario** | 01: The Enthusiast's Round |
| **Page Number** | 01.1 |
| **Platform** | Mobile |

## Overview

**Page Purpose:** Dashboard of sessions — find the one that needs attention and tap into it. This is the cold-open fallback; the primary Enthusiast path (push notification) bypasses this page entirely and lands on the Tape Submission page with search focused.

**User Situation:** Late 20s, on the couch after dinner, sees a notification that a new tape prompt dropped. Immediately starts thinking about the perfect song. They've been in this session for a few weeks — this is their ritual.

**Mental State:**
- **Hope:** Find the perfect song that says something about them, discover what friends picked, experience the reveal moment.
- **Worry:** Friction breaking the flow — too many steps between "I know my song" and "submitted."

**Discovery Method:** Push notification (primary — bypasses this page) or cold app open (secondary — lands here).

**Entry Context:**
- **Primary path:** Push notification → Tape Submission page directly (skips Session Home)
- **Secondary path:** Cold app open → Session Home

**Exit Action:** Taps a session card → Tape List

## Design Decisions

- **Notification deep-link:** Push notification for a new tape prompt takes the user directly to the Tape Submission page (01.3) with the prompt visible and search input focused. Zero taps to start typing a song name.
- **Session Home as safety net:** Only seen on cold opens — when a notification was dismissed, lost, or the user is checking in between prompts.
- **Multi-session support:** Users can belong to multiple sessions (e.g., college friends, coworkers). Each session card shows the current tape prompt and user-specific status.
- **Card-based layout:** Session cards in a grid (desktop) or vertical stack (mobile). No columns, no accordions, no grouping — just cards.
- **Own visual identity:** Light theme default with dark/light toggle. Not OHM's dark-first aesthetic — Anonymix has its own personality.
- **Tech stack shared:** OAuth, backend storage, progressive state patterns from OHM/toolbox. UI identity is distinct.

## Content Needs

- List of sessions the user belongs to
- Each session card layout (top to bottom):
  1. **Session title** (left) | **drag handle** (right) — justify-between
  2. **Avatar circles** — overlapping, max spacing by default, compress overlap if approaching full card width. No truncation.
  3. **Current tape prompt** — ellipsis truncated to one line
  4. **Action row:** [View Session] outlined button (left) | **status + deadline** two-line center block (status label top, countdown below, vertically centered) | **[Action button]** filled if applicable (right) — justify-between
- Status states with color coding: Submitting (green), Voting (amber), Playlist Ready (blue), Results (purple)
- Action buttons only on actionable states (Submit, Vote) — results/waiting states show badge only

## Driving Forces Addressed

- **Social discovery (14-15):** Glanceable status across friend groups — see what's happening in each session at a glance
- **Friction elimination (14-15):** Notification path skips this page entirely; cold-open shows exactly what needs attention

## On-Page Interactions

- Scan session list, tap a session card — minimal interaction
- No filters, no search, no settings on this page

## Complexity Notes

- Low complexity — card grid/stack with status variants per card
- No storyboard items, no modals, no complex states
- Responsive: grid on desktop, stacked cards on mobile — straightforward breakpoint
