## Overview

**Page Purpose:** Read the prompt, search for a song, submit it. This is the core action of the entire product — the moment the user expresses themselves through a song choice. Song Search (formerly 01.4) is merged into this page as an on-page interaction.

**Entry Context:**
- **Primary path:** Push notification → lands here directly, search input focused, keyboard up
- **Secondary path:** Tapped active tape from Tape List (01.2) or Session Home card

**Exit Action:** Taps Submit → returns to session view looking at the active tape. Can re-enter to clear/change submission before deadline.

## Design Decisions

- **Song Search merged in:** No separate search page. Search is an on-page interaction — autocomplete in the search input.
- **Two-stage selection:** Tap autocomplete result → fills input, drops keyboard, shows song preview (artist + title in input, album art below). User can clear to re-search or tap Submit. Not auto-submit — gives a beat to reconsider.
- **Cross-platform availability:** Songs not available on all music platforms used in the session are grayed out and disabled in autocomplete results. Keeps browsing intact while making the constraint visible.
- **Post-submit flow:** Submit → animate back to session view on the active tape. User can re-enter to clear/change before deadline.
- **Last-to-submit moment:** When this user is the last to submit, show a tape winding animation with countdown. Countdown leads to submission closing and playlist being finalized.

## Content Needs

- Tape prompt displayed prominently at top
- Search input (focused on notification entry, autocomplete from music platform API)
- Autocomplete results with cross-platform availability indicator (gray out unavailable)
- Selected song preview: artist + song title in input, album art below
- Submit button (visible after song selected)
- Submission status / current pick (when re-entering before deadline)

## Driving Forces Addressed

- **Friction elimination (15):** Notification → focused search → type → select → submit. Minimal taps, zero navigation.
- **Self-expression (hope):** The prompt + song choice is the creative moment — the core of what makes Anonymix meaningful

## On-Page Interactions (Storyboard Items)

- Search autocomplete with cross-platform availability filtering (gray out / disable)
- Tap result → fills input, drops keyboard, shows album art preview
- Clear input → re-focus search, keyboard up
- Submit → animate back to session view on active tape
- Last-to-submit: tape winding animation with countdown → submission closes → playlist finalized
- Re-entry before deadline: see current submission, clear to change

## Complexity Notes

- High complexity — cross-platform song availability check, music API integration, last-submitter detection
- Technical unknown: playlist generation approach may differ per platform (Spotify API vs YouTube Music API)
- Multiple storyboard items requiring animation/interaction design
- Responsive: mobile is the primary device; desktop would need different search input treatment

## Open Questions

- Playlist generation technical approach — likely different per platform
- Grace period: fixed countdown after last submit, or immediate close?
- Cross-platform search: constrain search to intersection of both catalogs, or search user's platform and validate against others on selection?
